# Code of Conduct

## Our Commitment

We want Editron to be a respectful, inclusive, and constructive open-source community for users, contributors, mentors, and maintainers.

Everyone participating in this project is expected to help create an environment that is:

- welcoming
- technically rigorous
- respectful during disagreement
- safe for newcomers and experienced contributors alike

## Expected Behavior

Examples of behavior that contribute to a healthy community:

- being respectful in language and tone
- giving constructive feedback
- assuming good intent before escalating
- accepting review comments professionally
- helping others understand project context
- focusing discussion on code, behavior, and documented requirements

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, insults, or personal attacks
- discriminatory or exclusionary language
- trolling or intentionally disruptive behavior
- publishing someone else's private information without permission
- hostile review behavior
- repeated bad-faith arguments after a maintainer decision is clearly documented

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- code reviews
- project discussions
- any public spaces where people represent the Editron project

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.

They may take any action they consider appropriate in response to behavior that violates this policy, including:

- warning
- comment moderation
- temporary participation restrictions
- permanent removal from project spaces

## Reporting

If you experience or witness unacceptable behavior:

- contact the maintainer through GitHub
- include links, screenshots, and context when possible
- do not escalate sensitive reports publicly unless necessary for immediate safety

All reports will be reviewed in good faith.

## Maintainer Standards

Maintainers should:

- apply rules consistently
- explain moderation decisions when appropriate
- separate technical disagreement from personal judgment
- protect contributors from harassment and dogpiling
